### Описание задачи:

Необходимо разработать таблицу в формате Single Page Application (SPA), которая будет отображать данные из базы данных и обеспечивать пользователю функциональность для сортировки, фильтрации и пагинации данных. Таблица должна работать без перезагрузки страницы, предоставляя пользователю удобный и интерактивный интерфейс для взаимодействия с данными.

 

### Задача:

 

**Требования к таблице:**


1. **Структура таблицы:** 
   - Таблица должна содержать 4 колонки: 
     - Дата
     - Название
     - Количество
     - Расстояние
2. **Функциональность таблицы:** 
   - **Сортировка:** 
     - Таблица должна иметь возможность сортировки по всем полям, кроме даты.
   - **Фильтрация:** 
     - Таблица должна содержать два выпадающих списка и текстовое поле для фильтрации данных: 
       - Выбор колонки для фильтрации.
       - Выбор условия фильтрации (равно, содержит, больше, меньше).
       - Поле для ввода значения для фильтрации.
   - **Пагинация:** 
     - Таблица должна содержать функциональность для пагинации данных.
     - Пагинация должна быть реализована на стороне сервера.
3. **Автоматическое заполнение данных:** 
   - При старте приложения база данных должна автоматически заполняться случайными значениями для демонстрации функциональности.
4. **Контейнеризация и развертывание:** 
   - Проект необходимо завернуть в Docker для облегчения развёртывания и обеспечения переносимости.
   - В проекте необходимо настроить Nginx для проксирования общения между фронтендом и бекендом.

 

**Технологии и инструменты:**

- **Backend:** 
  - Python 3.11
  - Django REST Framework для создания серверного API
  - PostgreSQL для хранения данных
  - Django ORM для работы с базой данных
- **Frontend**: 
  - React или Vue.js для создания интерактивного интерфейса
  - Axios для выполнения HTTP-запросов к серверному API
  - CSS библиотеки для стилизации (например, Bootstrap)
- **Проксирование:** 
  - Nginx для проксирования запросов между фронтендом и бекендом

 

**Ограничения:**

- Нельзя использовать библиотеки с готовыми компонентами или плагины для React/Vue, которые предоставляют готовый функционал, требуемый в задании.
- Нельзя использовать библиотеки и плагины для валидации.
- Нельзя использовать CMS системы.

 

### Технические требования:

1. **Фреймворк:** 
   - Backend: Django + DRF
   - Frontend: React или Vue.js
2. **База данных:** 
   - PostgreSQL для хранения данных
3. **ORM:** 
   - Использовать Django ORM для взаимодействия с базой данных
4. **Контейнеризация:** 
   - Docker для контейнеризации приложения
5. **Проксирование:** 
   - Nginx для проксирования запросов между фронтендом и бекендом
6. **Документация:** 
   - В корне проекта должен быть файл README.md с описанием структуры проекта и инструкциями по установке и запуску
   - Реализовать автогенерируемую документацию API с использованием Swagger
7. **Качество кода:** 
   - Соблюдать стандарты PEP8 для Python кода
   - Весь код должен храниться в удаленном Git репозитории
8. **Тестирование:** 
   - Написать тесты для всех основных функций платформы
   - Покрытие тестами должно быть минимум 75%
9. **Безопасность**: 
   - Настроить CORS для работы с доверенными доменными именами или IP адресами
